## 这个技能能做什么

- ✅ **从素材到剧本**：给定题材，自动搜集真实素材并戏剧化提炼。
- ✅ **三幕/四幕结构大纲**：生成完整的"幕-场-节拍"分层大纲。
- ✅ **3000 字以上深度单场**：采用 Beat-by-Beat 分节拍扩写，保证每场戏的厚度。
- ✅ **多人物冲突设计**：权力矩阵分析、利益碰撞设计、潜台词检验。
- ✅ **场间因果连贯性**：自动检验上下场之间的"因果锁"是否完整。
- ✅ **De-AI 文学抛光**：最终交付前，可选择去除 AI 味，提升文本的自然文学感。

---

## 六步工作流简介

```
/start   →  初始化项目，搜集素材
/outline →  生成全案大纲
/plan    →  拆解某一场的节拍
/write   →  正式写作（≥2500字/幕）
/compile →  合并所有幕次成完整剧本 ⭐ 【新增】
/review  →  导演质检
/de-ai   →  去AI味文学抛光（可选）
```

## 如何开始一个新剧本

### 标准提问模板（复制填写）

```
/start

剧名/简述：[写你的故事一句话简介，如：一个关于反腐的主旋律题材话剧]
主要主题：[宏观命题，如：反腐、人性救赎、命运抗争]
次要主题：[具体冲突，如：势力博弈、师徒决裂、家族恩怨]
登场人物及矛盾：
  - [人物A]：[身份 + 欲望 + 与谁的冲突]
  - [人物B]：[身份 + 欲望 + 与谁的冲突]
素材来源：[如：上网搜索真实案例 / 使用以下背景材料…]
全剧幕数：[建议：四幕]
```

**填写示例：**
```
/start

剧名/简述：一个关于反腐的主旋律题材话剧
主要主题：反腐（权力的腐蚀与法律的重塑）
次要主题：势力之间的斗争、师徒关系的决裂
登场人物及矛盾：
  - 周永盛（腐败局长）：试图掩盖受贿事实，对抗纪委；同时被上层势力操控
  - 林剑（纪检组长，周的学生）：试图让师傅认罪，同时承受情感与职业的双重压力
  - 孙建国（幕后大老虎）：冷静切割、弃卒保车
素材来源：上网搜索真实反腐案例
全剧幕数：4场
```

## 每个指令的使用说明

### `/start` — 启动项目
**何时使用**：开始一个全新的剧本。
**你需要提供**：剧名、主题、人物关系、素材方向、场次计划。
**系统会做**：搜集素材 → 提炼冲突点 → 建立人物矩阵 → 初始化记忆文件。

---

### `/outline` — 生成全案大纲
**何时使用**：`/start` 完成后，进入大纲阶段。
**你需要提供**：无，系统基于 `/start` 阶段的素材自动生成。
**系统会做**：生成幕结构 + 每场节拍列表 + 人物弧光 + 主题对位图。
**你需要做**：审阅大纲，确认后方可进入写作阶段。

---

### `/plan [场次]` — 场次细节规划
**何时使用**：大纲确认后，准备写第 N 场前。
**使用方式**：`/plan 第一场` 或 `/plan 第三场`
**系统会做**：为该场次生成人物任务矩阵、权力转移路径、每个节拍的具体规划。
**你需要做**：审阅规划，确认合理性，然后发送 `/write` 开始正式写作。

---

### `/write [场次]` — 正式写作
**何时使用**：场次规划已确认后。
**使用方式**：`/write 第一场`
**系统会做**：
- 按幕次顺序完整扩写，单幕 ≥ 2500 字
- 确保幕内内容连贯，无节拍分割感
- 全幕完成后执行人物逻辑三检（最高任务 / 信息管控 / 因果钩子）
**你需要做**：每幕完成后确认内容，然后继续下一幕。

---

### `/compile` — 幕次合并与完整剧本生成 ⭐ 【新增】
**何时使用**：所有幕次脚本都已通过 `/write` 完成后。
**使用方式**：`/compile 剧名`（如：`/compile 火候`）
**系统会做**：
- 自动搜索并识别所有幕次文件
- 验证幕次编号连续性（1-2-3-4...）
- 智能合并幕与幕之间的内容，去除重复标记
- 生成包含创作信息头、人物表、使用说明的完整剧本文件
- 自动执行格式检查（场景头、舞台指示、对话格式）
- 输出详细的合并质检报告
**输出物**：`[剧名]_完整舞台剧本_[时间戳].md`（可直接分发给导演/演员）
**详细说明**：查看 `config/DRAMA_COMPILE_GUIDE.md`

---

### `/review` — 导演质检
**何时使用**：某场初稿完成后。
**系统会做**：
- 检查台词的"广播剧腔"（On-the-nose 测试）
- 核查冲突密度（每 500 字一次权力转移）
- 验证场间因果钩子的完整性
**输出物**：审计报告 + 修改建议方向。

---

### `/de-ai` — 去 AI 味抛光（可选）
**何时使用**：`/review` 完成后，你认为文本需要更自然的文学质感时。
**系统会做**：
- 清除"而是/本质上/不仅…更是"等 AI 高频连接词
- 压缩冒号用量，强化动词具体性
- 保真改写（不增删信息，只做语言层面的"祛腔调"处理）
**输出物**：`[剧名]_第[N]场_DeAI纯净版.md`

## 常见工作流示例

### 场景 A：从零开始创作一个全新剧本
```
/start → 填写提问模板 → 确认素材摘要
     ↓
/outline → 审阅大纲 → 确认通过
     ↓
对每一幕循环：
  /plan 第一幕 → 确认幕次规划
  /write 第一幕 → 完成该幕写作
  
对每一幕循环：
  /plan 第二幕 → /write 第二幕
  /plan 第三幕 → /write 第三幕
  /plan 第四幕 → /write 第四幕
     ↓
/compile → 自动合并所有幕次成完整剧本 ⭐
     ↓
/review → 查看导演审计报告
     ↓
/de-ai → 去AI味文学抛光（可选）
```

### 场景 B：已有素材，只想写某一场
```
先告诉系统背景：
"这是一个关于[主题]的话剧。请直接 /plan 第二场。以下是人物信息：[...]"
/plan 第二场 → /write 第二场 → /review → /de-ai
```

### 场景 C：对已有剧本的某一幕进行深度扩写
```
"请对以下初稿进行深度扩写，要求单场 ≥ 3000 字，按 Beat-by-Beat 方式推进：[粘贴初稿]"
/review → /de-ai
```

## 常见问题

**Q：每场戏一定要 3000 字吗？**
A：是的。3000 字是保证单场戏具备足够冲突密度与心理厚度的最低标准。为了质量，系统会采用 Beat-by-Beat 分段写作，你可以每 2 个节拍后进行方向调整。

**Q：每次写完一场，下次可以继续吗？**
A：可以，且推荐每场写完后发送 `/review` 和可选的 `/de-ai`，然后更新记忆文件，下次直接从下一场的 `/plan` 继续。

**Q：可以写爱情剧、悬疑剧等非主旋律题材吗？**
A：完全可以。`/start` 时只需填写对应的主题和人物，系统会自动适配。Drama Genius 支持所有话剧题材。

**Q：De-AI 步骤是强制的吗？**
A：不是。`/de-ai` 是可选步骤，仅在你认为文本有明显 AI 播音腔时使用。对于风格化、实验性的写作，可以跳过。

**Q：如何修改已生成的场次？**
A：直接告诉系统："请修改第二场的节拍三，我希望周永盛的崩溃点发生在妻子的照片出现时，而不是在听到孙建国的名字时。"系统会在保持其他节拍不变的前提下进行局部改写。
