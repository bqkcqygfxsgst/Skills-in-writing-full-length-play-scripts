# 话剧项目上下文 (CLAUDE.md) v2.0

## 项目信息
- **项目名**：话剧剧本写作中心（Drama Genius）
- **技能版本**：v2.0（含 De-AI 集成工作流）
- **核心文件层级**：
  ```
  DRAMA_GENIUS_SKILL.md  ← 主技能定义（六步工作流 + 创作准则）
  SOUL.md                ← 艺术风格底座（斯坦尼 + 契诃夫 + 麦基）
  MEMORY.md              ← 全剧记忆（每场写完必须更新）
  ```

## 六步工作流（最新版）
1. **`/start`**：初始化项目，搜集素材，建立 MEMORY.md。
2. **`/outline`**：生成全案大纲（结构路线图 + 节拍分布 + 主题对位）。
3. **`/plan [场次]`**：拆解指定场次（人物任务矩阵 + 权力路径 + 节拍规划）。
4. **`/write [场次]`**：逐幕扩写（单幕 ≥ 2500 字，每幕完成后汇报进度）。
5. **`/compile`** ⭐ **【新增】**：合并所有幕次舞台脚本，生成完整演出剧本。
6. **`/review`**：虚拟导演质检（去广播剧化 + 冲突密度 + 因果钩子）。
7. **`/de-ai`**（可选）：用户确认后，调用 De-AI 技能对文本进行去AI味抛光。

## 规范锚点
| 规范类型 | 参考文件 |
| :--- | :--- |
| 舞台格式强约束 | `.claude/rules/drama-specs.md` |
| 艺术灵魂与风格 | `SOUL.md` |

| 戏剧理论参考 | `knowledge/` 目录下的知识库文件 |

## 自动化钩子
- **开场钩子**：每次开始新会话，优先读取 `MEMORY.md`，恢复当前剧本的进度、人物状态和未竟钩子。
- **结场钩子**：每场写作完成后，必须更新 `MEMORY.md` 中"场景进度与伏笔"栏，记录新增的悬念钩子。
- **状态保护**：若当前 MEMORY.md 中显示某场"已完成 DeAI 处理"，则跳过该场直接进入下一场规划。

| :--- | :---: | :--- |
| `/start` | 否（素材搜集，不涉及理论） | — |
| `/outline` | **是** | `"[主题] 三幕结构 人物弧光"` |
| `/plan` | 可选 | `"[场次类型] 权力转移 节拍设计"` |
| `/write` | **是** | `"[本场核心冲突] 潜台词 动作外化"` |

| `/de-ai` | 否（使用专属 De-AI 技能） | — |

### 调用方式
```bash
python scripts/drama_rag.py --query "查询关键词"
```

### 使用要求
1. **首次使用前**必须先建立索引：`bash scripts/setup_rag.sh`
2. 检索结果以 Markdown 格式返回，直接作为写作/大纲提示的**理论参考段落**注入。
3. **禁止全量加载** `knowledge/` 目录文件（文件总大小 ~1MB，会大量消耗 token）；只通过 RAG 按需取用。
4. 如检索返回"向量库为空"，提示用户运行 `bash scripts/setup_rag.sh`。

- **输出文件**：`output/[剧名]_素材库.md`

### 调用时机（强制）

```bash

```

| 步骤 | 是否调用搜索 | 示例 |
| :--- | :---: | :--- |
| `/start` | **是** | `--search "反腐剧" --name "权力对峙"` |
| `/outline` | 否（使用 `/start` 的输出） | — |
| `/plan` | 否 | — |
| `/write` | 否 | — |

### 快速开始
```bash
# 演示模式（立即查看输出格式）
python scripts/drama_search.py --demo

# 真实搜索（需要 Bocha API Key）
python scripts/drama_search.py --search "反腐剧" --name "XXX" --verbose
```

### 配置要求
在 `.env` 中配置：

```env
# 必需（已默认配置）
DASHSCOPE_API_KEY=sk-xxx...

# 可选（中文搜索增强）
BOCHA_API_KEY=your_bocha_key_here
```

### 使用要求
1. **首次使用**推荐先运行 `--demo` 模式验证工作流
2. 搜索脚本会自动处理 API 失败（回退到其他源）
3. LLM 智能提取基于 DashScope（通义千问），需要有效的 DASHSCOPE_API_KEY
4. 输出文件自动保存至 `output/` 目录，不需要手动保存

### AI 提取规范
- **冲突点**：从素材识别戏剧张力最强的 3 个对立
- **人物动机**：提取隐性心理驱动力（非表面行为表述）
- **动作瞬间**：具有视觉冲击力且暴露心理状态的 5 个动作

详见 [DRAMA_SEARCH_GUIDE.md](DRAMA_SEARCH_GUIDE.md)

### 大纲结构要求（话剧舞台版 ⚠️ 核心修改）

**重要声明**：以下规范已从"电影镜头思维"完全改为"话剧舞台思维"。

1. **故事简介**：用1-2句话清晰说明这个剧讲的是什么故事（核心人物、核心冲突、故事进展方向）

2. **剧目概览**：创作初衷 + 主题服务逻辑（如何通过次要主题推进主要主题）

3. **结构路线图**（三幕/四幕）- **按照舞台场景而非节拍组织**
   - 每幕分解为 **1-2个不同的物理空间（场景）**
   - 每个场景内包含 **6-8个节拍**（节拍是内部节奏参考，不是分割点）
   - **严禁**出现"第一场、第二场、第三场……"这样的多场景切割
   
   **结构示例**（错误）❌：
   ```
   第一幕：
   · 场景1：办公室（Beat 1-3）
   · 场景2：走廊（Beat 4-5）
   · 场景3：咖啡馆（Beat 6-8）
   ```
   
   **结构示例**（正确）✅：
   ```
   第一幕：主任办公室（一个连贯的故事空间）
   · Beat 1: 林晚进门，权力关系建立
   · Beat 2: 第一次冲突升级
   · Beat 3: 陈哲试图掌控局面
   · Beat 4: 林晚的反击
   · Beat 5: 信息翻转
   · Beat 6: 未竟事项（所有节拍在同一空间内完成）
   · [灯暗——幕落，如需换景最多30秒]
   
   第二幕：走廊+传达室（转换是戏剧内容的一部分）
   · Beat 7-12: 余波冲突最后在另一个象征空间完成
   ```

4. **人物欲望曲线**：开场欲望 → 人物障碍 → 最终变化

5. **主次主题对位图**：
   - 每一幕（而非每个微小场景）承载哪个次要主题
   - 不必详细到每个Beat，而是**以幕为单位**说明主题承载

6. **舞台视觉设定**：
   - 整体色调 + 核心意象（符号系统）
   - **舞台物理可实现性**：该剧需要几套布景？能否快速切换？演员数量？

### 输出验证（新增舞台实现性检查）
生成的大纲文件应满足：
- [ ] 故事简介已清晰表述：一句话能说清楚这个剧讲了什么
- [ ] **每幕场景数** ≤ 2 个（验证是话剧而非电影）
- [ ] **节拍总数**：三幕≥18个，四幕≥24个（节拍在单一或双重场景内完成）
- [ ] 人物任务：每位人物至少在3个关键节拍中出现
- [ ] 主题承载：每幕都明确对应一个或多个次要主题
- [ ] 冲突密度：每500字内至少一次权力转移或价值观碰撞
- [ ] **舞台可演出性**：①是否能用舞台道具实现？②演员换景时间是否需要>2分钟？③是否有超过10秒的无人空场？ 若有，改进方案是什么？

## 场景规划规范（新增）

### 架构说明
- **规划脚本**：`scripts/drama_plan.py`（场景级细化规划）
- **输入来源**：大纲 (`drama_outline.py` 输出)
- **输出文件**：`output/[剧名]_场景N_规划表.md`
- **规划内容**：人物任务矩阵 + 权力流路径 + 节拍定位 + 冲突密度验证 + 因果钩子

### 调用时机
仅在 `/plan` 步骤调用：

```bash
python scripts/drama_plan.py --drama "[剧名]" --scene [序号]
```

| 步骤 | 是否调用 | 示例 |
| :--- | :---: | :--- |
| `/outline` | 否 | — |
| `/plan` | **是** | `--drama "权力对峙" --scene 1` |
| `/write` | 否（使用 /plan 输出） | — |

### 规划表结构要求
1. **人物任务矩阵**：每个角色在本场的目标 + 障碍 + 具体行动
2. **权力流路径**：权力如何转移（谁从被动变主动、或相反）
3. **节拍定位**：本场包含的关键节拍序号 + 每个节拍承载的信息
4. **冲突密度检查**：是否满足 500 字/1 次冲突要求
5. **因果钩子**：本场结尾留给下一场的悬念事件

### 快速开始
```bash

python scripts/drama_plan.py --demo

# 实际规划第 1 场
python scripts/drama_plan.py --drama "权力对峙" --scene 1
```

## 场景写作规范（新增）

### 架构说明
- **写作脚本**：`scripts/drama_write.py`（完整幕级的连贯写作）
- **输入来源**：规划表 (`drama_plan.py` 输出) + 大纲
- **输出文件**：`output/[剧名]_第N幕_脚本.md`
- **写作原则**：**每一幕是不可分割的戏剧单位，节拍是内部节奏参考，不是内容分割**

### 核心写作理念

#### ⚠️ 节拍与连贯性的关系

**错误做法**：
```
### Beat 1: 场景 A
(场景头、人物、对话……)

### Beat 2: 场景 B  
(新的场景头、新的人物状态……)

### Beat 3: 场景 C
(又一个场景头、又一个人物转换……)
```
❌ **问题**：观众/读者会感觉到"跳跃"，失去沉浸感。场景头频繁重复且模式化。

**正确做法**：
```
[时间：X 点，场景A]
[地点：...]
[舞台布景：...]
[人物状态：...]

人物 A: (动作) 台词

人物 B: (动作) 台词
  (在同一场景内，人物的行动连贯，冲突逐步深化)

（时间推移，同一场景内发生的事件序列，通过场景转换自然过渡）

(人物走向另一个地点，这时才出现新的场景头)

[时间：Y 点，场景B]
[地点：...]
...
```
✅ **优点**：读者/观众感受到**连贯的故事流**，场景头只在必要时出现。

### 调用时机
仅在 `/write` 步骤调用：

```bash
python scripts/drama_write.py --drama "[剧名]" --scene [序号]
```

| 步骤 | 是否调用 | 示例 |
| :--- | :---: | :--- |
| `/plan` | 否 | — |
| `/write` | **是** | `--drama "权力对峙" --scene 1` |
| `/review` | 否（使用 /write 输出） | — |

### 脚本生成流程（改进版）

1. **加载规划表 + 大纲**：获取整幕的人物任务、权力流、节拍序列，以及世界设定、主题指向
2. **建立幕级架构**：理解整幕的开场欲望、中间冲突、最终落幕如何形成一个完整的戏剧弧线
3. **节拍→场景转换点的转译**：
   - 识别每个节拍代表的关键事件或冲突升级
   - 将相邻节拍之间的逻辑关系转化为**人物行动的因果链**
   - 只在**物理空间改变**时创建新的场景头，不在节拍之间硬生生插入
4. **整幕 LLM 写作**（推荐）：
   - 而不是逐节拍分别调用LLM
   - 给LLM提供完整的幕级上下文：人物、冲突、时间线、目标
   - 输出应该是一个**连贯的、有机的、可直接舞台演出的文本**
5. **后处理（移除节拍割裂）**：
   - 如果节拍标记（"### Beat 1"等）仍然存在，应该以隐形化或删除它们
   - 或者用场景转换来替代节拍标记（如：灯光的变化、时间的推移、地点的改变）
   - 最终的文本应该**看不出节拍痕迹**

### 快速开始
```bash
# 演示模式（查看脚本输出格式——注意连贯性）
python scripts/drama_write.py --demo

# 实际写作第 1 幕（完整的、连贯的剧幕呈现）
python scripts/drama_write.py --drama "权力对峙" --scene 1

# 写作第 2 幕
python scripts/drama_write.py --drama "权力对峙" --scene 2
```

### 输出验证

生成的脚本文件应满足：
- [ ] **完整性**：形成一个独立的戏剧弧线（开场-发展-高潮-落幕）
- [ ] **连贯性**：整幕内容流畅，人物情感线贯穿始终，无"跳跃感"
- [ ] **字数**：≥2500 字（四幕剧每幕建议2500字，保证总体10000+）
- [ ] **场景头**：仅在**地点改变时**出现，不在节拍之间频繁重复
- [ ] **场景头格式**：每个新地点都遵循四行格式（时间、地点、舞台布景、人物状态）
- [ ] **动作指示**：每 500 字内至少 5 个动作提示，且动作必须可观察+揭示心理
- [ ] **潜台词**：严禁出现人物直接"说出内心"的台词
- [ ] **冲突密度**：每 500 字内至少一次权力转移或价值观碰撞
- [ ] **因果钩子**：本幕结尾是否为下一幕埋入悬念或因果事件
- [ ] **节拍痕迹**：是否已隐形化或完全移除，使读者无感节拍分割

### 反面教材

❌ **这样的段落表示节拍分割过度**：
```
### Beat 1: 权力地图
[时间：下午3点，办公室]
[地点：主任办公室]
...（人物对话）

### Beat 2: 失望的信息
[时间：下午3点30分，同一办公室]
[地点：主任办公室]
...（人物转身走向门口）

### Beat 3: 门外的等待
[时间：下午3点40分，办公室门外]
[地点：办公室走廊]
...
```
**问题**：场景头在同一办公室内频繁重复，节拍分割打破了叙述的流畅性。

✅ **改进方案**：
```
[下午3点，主任的办公室内。...]

（主任讲述计划......）

（人物的情感在对话中逐步变化......）

（最后，他转身走向门口。）

（他步入走廊。灯光随之变化。）

[时间同样推移到下午3点40分，走廊内...]

（新的场景只在**必要时**才创建场景头）
```
