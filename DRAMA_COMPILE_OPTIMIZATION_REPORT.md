## 🎯 新增功能概览

### 核心功能：`/compile` 步骤

**位置**：第4.5步，位于 `/write` 和 `/review` 之间

```
/write → /compile ← 【新增】→ /review
```

### 主要功能点

1. **自动文件识别**
   - 自动搜索该剧目的所有幕次文件
   - 支持的文件名模式：
     - `[剧名]_第[N]幕_舞台剧本格式.md`
     - `[剧名]_第[N]幕_脚本.md`

2. **智能验证与合并**
   - ✓ 验证幕次编号连续性（1-2-3-4...）
   - ✓ 检查所有文件可读性
   - ✓ 去除幕间重复的"幕结束"标记
   - ✓ 保持舞台指示 `*(...)*` 和对话 `**Name**:` 的格式统一

3. **自动生成元数据**
   - 创作信息头（剧名、幕数、字数、演员需求等）
   - 完整的人物表
   - 使用说明和舞台美术建议
   - 编译信息和质量检查报告

4. **多格式支持**
   ```bash
   --format "stage"  # 标准舞台剧本格式（默认）
   --format "epub"   # 电子书格式（可选）
   --format "pdf"    # PDF打印格式（可选）
   ```

5. **完整的质检机制**
   - 文件完整性检查
   - 格式规范性验证
   - 内容一致性检查
   - 字数统计和统计报告

## 🎓 最佳实践指南

### ✅ 推荐做法

1. **完成所有幕次后再合并**
   ```bash
   /write 幕1 ✅ → /write 幕2 ✅ → /write 幕3 ✅ → /write 幕4 ✅ → /compile
   ```

2. **合并前检查格式**
   - 确保所有幕次文件名遵循规范
   - 确保幕编号连续（无跳号）

3. **利用时间戳版本管理**
   - 自动生成新的时间戳版本
   - 便于对比不同版本的变化

4. **直接发送完整剧本**
   - 导演和演员可直接使用
   - 无需额外的格式调整

### ❌ 避免做法

1. 不要手动编辑幕次编号
2. 不要在合并后修改单幕（应重新合并）
3. 不要混用不同的文件名格式
4. 不要用其他工具手动合并
