# Drama Genius 幕次合并功能 - 完整指南

## 📌 功能概述

**新增步骤**：`/compile` — 幕次合并与完整剧本生成

这是六步工作流中的**第4.5步**，位于 `/write`（逐幕写作）和 `/review`（虚拟导演质检）之间。

### 工作流位置

```
/start → /outline → /plan → /write → /compile ← 【新增】→ /review → /de-ai
```

---

## 🚀 核心功能

### 1. 自动搜索并识别所有幕次文件

**自动识别模式**：
- `[剧名]_第[N]幕_舞台剧本格式.md`
- `[剧名]_第[N]幕_脚本.md`

**验证机制**：
- ✓ 确保幕次编号连续（1-2-3-4...不能跳号）
- ✓ 检查所有文件都存在
- ✓ 按幕次序号自动排序

### 2. 智能合并与格式统一

**合并逻辑**：
- 按幕次序号顺序读取各文件
- **保留每幕的完整场景头**（时间、地点、布景、人物状态）
- **去除重复标记**：移除各幕末尾的"幕结束"标记
- **添加幕分隔符**：在幕与幕间插入可读的分隔线
- **格式一致性检查**：确保所有舞台指示保留 `*(...)*` 格式，对话保留 `**Name**:` 格式

**输出结构**：
```
【创作信息头】
  - 剧名、幕数、字数、演员需求、舞台难度、演出时长
  - 人物表、使用说明、舞台美术建议

【第1幕内容】
────────────────────────
【第2幕内容】
────────────────────────
【第3幕内容】
────────────────────────
【第4幕内容】

【尾声信息】
  - 编译时间戳、质量检查报告
```

### 3. 完整的元数据和指引

自动生成的信息头包含：

| 项目 | 说明 |
|:---:|:---|
| **基本信息表** | 剧名、幕数、总字数、演员需求、舞台难度 |
| **使用说明** | 文本格式规范、演员指引、舞台美术建议 |
| **人物表** | 主要人物、年龄、身份、关键特征 |
| **创作信息** | 创作背景、主要主题、次要主题、舞台性特点 |

### 4. 多格式支持

```bash
# 舞台剧本格式（常用）
python scripts/drama_compile.py --drama "火候" --format "stage"

# 电子书格式（可选）
python scripts/drama_compile.py --drama "火候" --format "epub"

# PDF格式（可选）
python scripts/drama_compile.py --drama "火候" --format "pdf"
```

## 🔍 质量检查机制

合并过程中自动执行以下检查：

### 1. 文件完整性
- [ ] 所有幕次文件是否都存在？
- [ ] 幕次编号是否连续（无跳号）？
- [ ] 每个文件是否可读？

### 2. 格式规范性
- [ ] 场景头是否都遵循四行格式？
- [ ] 舞台指示是否全部使用 `*(...)*`？
- [ ] 对话格式是否全部使用 `**Name**:`？
- [ ] 特殊符号是否正确？

### 3. 内容一致性
- [ ] 人物名称在各幕间是否保持一致？
- [ ] 是否有多余的"幕结束"标记残留？
- [ ] 幕与幕之间是否有无谓的空行？

### 4. 字数统计
- [ ] 是否正确统计了总字数？
- [ ] 单幕字数是否符合预期（通常 ≥ 2500 字/幕）？

**检查失败时**：脚本会列出具体问题及其位置，指导用户修正。

## ⚠️ 常见问题与解决方案

### Q1: 找不到幕次文件

**症状**：`❌ 未找到《火候》的任何幕次文件`

**原因**：
- 文件名不符合规范
- 文件位置不在 `output/` 目录
- 还未执行 `/write` 步骤

**解决**：
```bash
# 检查文件是否存在
ls output/ | grep "火候"

# 确保文件名格式正确
# 应为：火候_第1幕_舞台剧本格式.md (或 _脚本.md)
```

### Q2: 幕次编号不连续

**症状**：`❌ 幕次编号不连续`

**原因**：
- 某幕脚本还未生成
- 文件名中的幕编号有误（如：第01幕、第1场等）

**解决**：
```bash
# 补全缺失的幕次
python scripts/drama_write.py --drama "火候" --scene 3

# 检查文件名
# ✅ 正确：火候_第3幕_舞台剧本格式.md
# ❌ 错误：火候_第03幕_舞台剧本格式.md
```

### Q3: 格式检查失败

**症状**：`❌ 格式规范性检查失败`

**原因**：
- 某幕使用了异常的格式
- 舞台指示未使用 `*(...)*`
- 对话未使用 `**Name**:`

**解决**：
- 检查drama-specs.md的格式规范
- 修正该幕的格式问题
- 重新运行 `/compile`

### Q4: 字数统计不准

**症状**：统计字数与预期不符

**原因**：
- markdown格式符号被计入
- 某幕内容包含了原始代码块

**解决**：
- 脚本已自动去除markdown格式符号再统计
- 如仍有问题，手动清理各幕文件

## 🎓 最佳实践

### ✅ 推荐做法

1. **完成所有幕次后再合并**
   - 不要中途合并再处理
   - 确保所有格式规范都已到位

2. **合并前执行格式检查**
   - 使用 `/review` 对单幕进行审计
   - 确保格式100%规范再合并

3. **保留合并历史**
   - 每次合并生成新的时间戳版本
   - 便于版本追溯和对比

4. **使用统一的文件名**
   - 严格遵循 `[剧名]_第[N]幕_舞台剧本格式.md` 格式
   - 避免特殊符号和空格

### ❌ 避免做法

1. 不要手动编辑幕次编号
2. 不要使用特殊字符和空格在文件名中
3. 不要在合并后再修改单幕内容（应重新合并）
4. 不要用其他工具手动合并（可能导致格式混乱）
